按主题归档的个人知识库；左下角浮窗是局部图谱，可以拖拽放大；右上角放大镜可全文模糊搜索；按 `Ctrl/Cmd+K` 也能直接调出搜索面板。

## 元数据 / 维护工件

- [[失效黑名单]] — 已确认死链/灰色资源/账号迁移的黑名单，避免重复添加
- [[清理规则]] — 删除规则与分类指引（合并自原 待处理 / 待清理 三份规则文档）
- [[未分类长尾]] — 两轮自动分桶后仍未归类的 ~180 条长尾，需要人工判断
- [[清洗报告]] / [[精选报告]] — 早期批次的清洗与精选过程
- [[原始数据备份]] — urls2 批次的原始 markdown 备份
- [[总目录-链接归档|总目录·链接归档]] / [[链接整理归档]] — 早期批次的索引（仅参考）

## 维护规则

1. **新增链接**：先扔进对应主题文件的"长尾（未审核）"小节，确认长期价值后再上升到精选区或 [[个人常用版]] / [[知识清单]]
2. **失效链接**：抓取时如果显示 `403/404/账号已迁移/访客不可达/错误/超时/域名不可访问`，跳过主分类，只记入 [[失效黑名单]]
3. **专题膨胀**：主题文件超过 ~50 条精选时再拆子文件；同一主题保留集中度
4. **登录态页面**：需要 token 才能复现的页面归 **工作台**，不混入主题归档
5. **批次区分**：[[Unity-框架与工具]] 是早期 OneTab 批的官方框架/工具入口；[[Unity-UI与优化]] 是后续 urls2 批的 UGUI/PureMVC/优化文章
6. **批量分类**：定期运行 `python scripts/classify_cleaned.py` + `python scripts/refine_unsorted.py` 把 cleaned.txt 新增长尾分桶到对应主题
