## 一、删除规则

### 1. 搜索结果页

- Google / Google HK / 百度 / Bing 搜索结果
- ByteTech 搜索页
- Bilibili / B 站搜索结果

原因：搜索页只是"找资料的过程"，不是资料本身。

### 2. 会话页 / 邮件页 / 收件箱页

- `chatgpt.com/c/...`
- `gemini.google.com/app/...`
- `mail.google.com/...` / `mail.163.com/...`
- 带 `sid`、`session`、`token` 的邮箱或对话页面

原因：依赖登录态和一次性上下文，几乎不可复用。

### 3. 控制台深链接 / 日志页

- Jenkins `console` / `workspace` / `Temp_RunBuildStep*.log` 页
- Firebase / Google Cloud / Google Play Console 的细项设置页与问题页
- 带 `trace_event` / `lane_id` 的内部知识平台深链接

原因：时效性强，生命周期短。**只保留每个系统的根入口或高频首页一份**。

### 4. 灰色资源 / 破解 / 镜像下载

- 破解仓库 (StarUML / 各种 cracked)
- 非官方资源下载镜像
- Z-Library 镜像
- Unity 付费资产的第三方下载页
- 磁力 / BT / Telegram 资源频道

原因：不适合归档到正式知识库。

### 5. 本地文件路径 / 无语义页面

- `file:///...`
- 标题缺失、标题乱码、只剩系统默认标题的页面
- 抓取标题为 URL 自身的页面（说明站点没正常返回 OG title）

原因：无法形成稳定的检索入口。

## 二、内部页面分类规则

### 1. 长期知识（保留并入主归档）

判定关键词：`SDK`、`部署`、`架构说明书`、`接入备忘`、`证书`、`知识库使用需知`、`框架指南`、`引擎`、`引继流程`、`配置梳理`。

去向：[[内部长期项目页]]（如有需要从主归档点击进入）。

### 2. 项目执行中（单独存成项目看板，不并入主知识库）

判定关键词：`TODO`、`工作内容`、`研讨会`、`时间规划`、`需求`、`任务拆解`、`Hotfix 填写`、`数据服务切换`、`版本上线检查项`。

去向：[[失效黑名单]] 的"项目过程页"章节（这类只能登录态访问，公开 KB 没有价值）。

### 3. 临时入口 / 可删

判定关键词：`知识平台`（无具体主题）、`Docs`（无具体主题）、`drive/shared` / `drive/me`（云盘根入口）、带 `knowledgeId=` 的深链接、带 `state=RANDOMSTATE` 的授权页。

去向：[[失效黑名单]]。

## 三、保留代表入口的规则

同一站点同一主题出现多条链接时，**只保留 1 条最稳定的代表入口**，其余删除。常见情形：

- 知识平台类：保留 `知识平台首页` 一条
- 项目首页类：保留 `项目根入口` 一条，不留具体事项页
- 飞书 Wiki 同标题不同 sheet 的多条 → 合并保留主链接
- 微信公众号文章重定向：保留实际 article URL，去掉 `wechat_redirect` 跳转

## 四、人工复核后再删

下列类型不能机械删除，需要人工判断"是否真正用过"：

- 各类 CSDN 经验帖
- 零散知乎问答
- 未确认是否真正使用过的 GitHub Demo
- 只因一次故障排查而打开过的文章
- 标题模糊但 URL 看上去像官方页面的链接

保留标准：
- 是否是官方文档
- 是否是项目实际用过的方案
- 是否比搜索关键词更值得长期保存

## 五、删除顺序建议

1. 先删搜索页（最干净，无判断成本）
2. 再删会话页和邮箱页
3. 再删 Jenkins / 控制台深链接
4. 再删灰色资源和镜像下载页
5. 再删带 `knowledgeId=` 的内部深链接
6. 最后人工清理长尾低价值文章和 Demo

## 六、相关脚本

仓库 `scripts/` 下已有可复用的清理工具：

- `scripts/strip_lineage.py` — 剥掉文档里残留的 `urls`/`urls2`/`onetab_urls.14` 数据源前缀
- `scripts/clean_titles.py` — 删除 `onetab_urls.14[ -]<rest>` 这类标题前缀；同时去掉每篇正文第一行 H1 避免和 Quartz 自动渲染重复
- `scripts/add_frontmatter.py` — 给新建文件批量补 YAML frontmatter（title 取 H1，tags 按目录推断）
- `scripts/convert_to_wikilinks.py` — 把 markdown 内部相对链接 `[label](./xxx.md)` 转成 Quartz wikilink `[[xxx|label]]`
- `scripts/classify_cleaned.py` — 从 `cleaned.txt - selected.txt` 差集里按规则分桶到各主题长尾文件
